## Atualização Backend – 19/11/2025

### Contexto
Consolidamos o suporte RAG dos agentes EmployeeVirtual, criando o fluxo completo para upload de PDFs, persistência no MongoDB, integração com Pinecone e execução via PydanticAI. Este documento serve como referência rápida para a squad – especialmente para o Henrique (frontend) que consumirá as novas rotas ao integrar o upload e execução RAG no app.

### Principais Mudanças
- **Upload vetorial dos agentes** (`POST /api/agents/{agent_id}/documents`)
  - Aceita `filepdf` (binário do PDF) e campo opcional `metadone` com JSON.
  - O backend processa o PDF localmente (`VectorStoreService`), gera embeddings com OpenAI, cria namespace por agente e grava tudo direto no Pinecone (sem microserviço externo).
  - Registra cada upload em `Collections.AGENT_DOCUMENTS` (*MongoDB*), guardando metadados e payload retornado pelo Pinecone.
- **Execução RAG**
  - `AIService` agora inicializa o `RagAgentRunner` (OpenAI + Pinecone) quando `PINECONE_*` estiverem configurados.
  - `AgentService.execute_agent` detecta se o agente possui documentos e utiliza o fluxo RAG (tool `retrieve` com Pinecone + embeddings) antes de responder.
  - Fallback automático mantém o comportamento anterior quando não há corpus ou chaves de Pinecone.
- **Cliente Pinecone**
  - Criado `VectorStoreService` interno para extrair texto de PDFs, gerar embeddings e fazer upsert em namespace dedicado do agente.
  - `RagAgentRunner` continua usando tool `retrieve` que consulta o índice Pinecone filtrando por `agent_id`/namespace.

### Novas Variáveis de Ambiente
| Variável | Descrição | Default |
|----------|-----------|---------|
| `PINECONE_API_KEY` | API key Pinecone utilizada pelo `RagAgentRunner` | _None_ |
| `PINECONE_INDEX_NAME` | Nome do índice Pinecone (ex.: `employeevirtual-agents`) | _None_ |
| `PINECONE_CLOUD` | Cloud provider do índice serverless (`aws`, `gcp`, etc.) | `aws` |
| `PINECONE_REGION` | Região do índice serverless | `us-east-1` |

> **Importante**: certifique-se de definir `OPENAI_API_KEY`, `PINECONE_API_KEY` e `PINECONE_INDEX_NAME` no `.env` para que o RAG seja habilitado. Sem esses valores o sistema volta automaticamente ao modo “não-RAG”.

### Para o Henrique (Frontend)
- Consumir a nova rota `POST /api/agents/{agent_id}/documents` enviando:
  - `filepdf`: arquivo `multipart/form-data`.
  - `metadone`: string JSON com metadados extras para enriquecer a busca (ex.: `{"curso":"IA","modulo":"1"}`).
- Após o upload, o backend retornará o registro salvo no Mongo (`document`) e a resposta do microserviço vetorial (`vector_db_response`). Use esse retorno para exibir status e permitir gerenciamento de documentos.
- Durante a execução do chat/agente, exibir indicação visual quando o backend responder com contexto RAG (opcional, mas ajuda usuários a entenderem que o conhecimento vem do PDF).
